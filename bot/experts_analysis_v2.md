## 🔍 一、数据专家视角分析

### 1.1 数据架构现状

#### ✅ 优点
| 维度 | 说明 |
|------|------|
| **多重数据源降级** | 指数 4 层降级（新浪→腾讯→东方财富→历史数据），个股 3 层，ETF 3 层 |
| **数据质量监控** | `core/data_quality.py` 实现完整性监控，含 5 维度评分（完整性/准确性/一致性/时效性/有效性）|
| **缓存策略精细** | 7 个模块独立 TTL（市场 30min、宏观 12h、北向 30min、ETF 30min、龙头 30min、全球 1h、北证 30min）|
| **数据验证** | `_validate_index_value()` + `DataValidator` 历史一致性校验 |

#### ⚠️ 问题与风险

1. **同步调用阻塞事件循环**（P1）
   - `data_fetcher.py` 中的 `akshare` 调用是同步的，虽然文档注释说"通过 `run_in_executor` 在线程池中执行"，但实际代码仍直接调用同步函数
   - 建议：使用 `asyncio.to_thread()` 或 `loop.run_in_executor()` 包装所有 akshare 调用

2. **数据质量报告未对接 LLM 解读**（P2）
   - `data_quality.py` 生成了高质量的数据质量报告，但 LLM prompt 中完全没有考虑数据质量信息
   - 大师兄解读无法区分"数据真的差"和"数据获取失败"的场景

3. **缺失的数据维度**（P3）
   - **北向资金持仓结构**（行业分布、十大持仓）缺失 — 这是大师兄体系中的"八大不平衡"分析的重要输入
   - **两融余额**缺失 — 融资融券数据可辅助分析市场情绪
   - **股指期货基差**缺失 — 反映机构投机头寸

### 1.2 改进建议

```python
# 示例：将数据质量信息嵌入 LLM 调用
async def generate_commentary(market_summary: str) -> str:

    )
```

### 2.2 ✅ 架构优点

1. **分层清晰**：`config/core/services/app/models` 五层分离，各司其职
2. **配置管理规范**：Pydantic `BaseSettings` + `.env` 文件，类型安全
3. **缓存设计合理**：模块级 TTL + 文件缓存 + 自动过期清理（48h）

5. **错误处理完善**：`@retry` 装饰器 + 多重数据源 + 熔断模式（`monitor.should_skip`）

### 2.3 ⚠️ 核心架构问题

#### 🔴 P0: LLM System Prompt 硬编码（最严重问题）

`llm_service.py` 第 14-56 行的 `MASTER_XIONG_SYSTEM_PROMPT` 是直接硬编码在 Python 文件中的约 50 行文本，而本地已有 `SKILL.md v2.0.0`（399 行，约 19KB），两者对比：

| 维度 | 硬编码 Prompt (v1) | SKILL.md v2.0.0 |
|------|:------------------:|:----------------:|
| **行数** | ~50 行 | 399 行 |
| **内容量** | 约 1.8KB | 约 19KB（10x） |
| **八大不平衡** | ✅ 有（简短列举） | ❌ 未直接包含（需补充） |
| **三层传导元方法论** | ✅ 有（A股专用版 3 层） | ✅ 有（10 种变体，贯穿一切领域） |
| **清净心本体论** | ❌ 无 | ✅ 83 行详细哲学体系 |
| **佛学修行** | ❌ 无 | ✅ 32 行（了凡四训、八识海） |
| **宏观经济体系** | ❌ 无 | ✅ 68 行（康波、金油、资本史） |
| **黑盒子估值** | ❌ 无 | ✅ 32 行 |
| **全球资本史** | ❌ 无 | ✅ 20 行（14 世纪-现代） |
| **投资智慧** | ✅ 有（5 条原则） | ✅ 有（反大众共识、股市如血脉） |
| **历史视角** | ❌ 无 | ✅ 35 行（郑和 vs 麦哲伦、朝代更替） |
| **生活智慧** | ❌ 无 | ✅ 25 行（苏东坡留余、焦虑症） |
| **概念词典** | ❌ 无 | ✅ 18 个关键概念定义 |
| **可维护性** | ❌ 必须改代码 | ✅ 独立文件、支持热加载 |
| **版本管理** | ❌ 不可追踪 | ✅ Git 可追踪变更 |

**影响**：当前大师兄解读只能基于有限的 50 行框架进行分析，无法调用完整的知识体系进行深度剖析。例如：
- 无法进行"金油汇债G"五维度交叉分析
- 无法从"三层传导"元方法论角度解释市场传导机制
- 无法从"康波周期"和"资本史"角度进行长周期定位
- 无法使用"黑盒子估值"框架分析龙头股

**建议架构改造**：

```python
            return f.read()
    logger.warning(f"SKILL.md not found at {SKILL_PATH}, using built-in prompt")

# 替代方案：配置化路径
```

## 👤 五、UX 专家视角分析

### 5.1 飞书卡片体验评估

#### ✅ 优点

- **信息结构清晰**：异动提醒 → 行情 → 宏观 → 北向 → ETF → 龙头 → 全球 → 解读

- **时间戳展示**：底部显示数据更新时间

#### ⚠️ 用户体验问题

##### 🟡 P1: 大师兄解读缺少"人格化"风格

当前生成的解读虽然使用框架，但缺少 SKILL.md 中"悟而后醒"的独特分析风格：
- 缺少"犀利、直击本质、用通俗语言解释复杂逻辑"的个人特征
- 缺少"三层传导"元方法论的层次感
- 缺少生动的案例类比（如"南京猪肉"、"双龙戏凤"）

##### 🟡 P2: 飞书卡片信息过载

详情卡片包含 9 个模块 + 异动提醒 + 大师兄解读，信息密度极高。用户可能：
- 在手机上需要大量滚动才能看完
- 难以快速定位自己关心的信息

**建议方案**：
```
方案 A：分卡片推送
- 摘要卡：核心指数 + 异动 + 大师兄解读（精简版）
- 详情卡：仅包含其他模块（可折叠设计）

方案 B：Tab 式设计
- 一个卡片内使用飞书的 Tab 可切换模块
- 目前飞书卡片支持 Tab 元素
```

##### 🟢 P3: 缺少交互功能

- 用户无法对日报进行反馈（"这次的解读有用吗？"）
- 无法订阅特定模块或设置提醒阈值
- 缺少"展开更多"的渐进式信息展示

### 5.3 优化建议

```
1. 大师兄解读增加人格化前缀
   例如："大师兄看盘 | 2026-06-12"
   → 使用三层传导框架：今天资金从消费电子传导到...
```

```python
# 分层 Prompt 组装
class MasterXiongPrompt:
    BASE = "你是'大师兄'..."
    SKILL_CORE = load_skill_sections(["三层传导", "八大不平衡", "黑盒子估值", "宏观体系"])
    WEATHER_ANALYSIS = "今日关注：北向资金异动"
    
    @classmethod
    def build(cls, market_data: dict) -> str:
        # 根据当日市场特征，动态选择最相关的 SKILL 章节
        selected = [cls.BASE]
        if has_north_flow_alert(market_data):
            selected.append(cls.SKILL_CORE["资本流动不平衡"])
        if has_sector_rotation(market_data):
            selected.append(cls.SKILL_CORE["三层传导-板块轮动"])
        return "\n\n".join(selected)
```
