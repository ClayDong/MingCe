## 📊 二、数据专家视角分析

### 2.1 数据为空的核心原因

从之前的缓存分析可以看到：
```json
{
  "indices": [
    {"name": "上证指数", "value": 4056.6775, "change_pct": -0.027, "volume": "0"},
    {"name": "科创50", "value": 1737.9654, "change_pct": -0.005, "volume": "0"},
    {"name": "深证成指", "value": 0.0, "change_pct": 0.0, "volume": "0"},
    {"name": "创业板指", "value": 0.0, "change_pct": 0.0, "volume": "0"}
  ],
  "up_count": 0, "down_count": 0, "flat_count": 5524,
  "total_volume": "",
  "top_sectors": [], "bottom_sectors": []
}
```

**问题诊断：**

1. **数据获取失败导致缓存旧数据**
   - AKShare 数据源可能被反爬虫拦截
   - 或者请求超时
   - 但缓存仍保存了旧的、无效的数据

2. **缺少数据质量验证和降级机制**
   - 虽然代码有 `_validate_index_value` 函数，但在获取数据时是否应用了降级？
   - 需要检查 data_fetcher.py 中多重数据源的实现

3. **涨跌统计异常**
   - up_count 和 down_count 都是 0，所有股票都是平盘
   - 很可能是个股数据源请求失败

### 2.2 数据专家建议

1. **增加数据源熔断机制**
   - 当某个数据源连续失败 N 次，暂时禁用一段时间
   
2. **数据获取前检查交易时间**
   - 非交易时间可以不获取实时行情，而是获取最近一个交易日的收盘数据
   
3. **更健壮的数据验证**
   - 在存入缓存前，严格检查数据是否有效
   - 无效数据不存入缓存，继续尝试备用源
